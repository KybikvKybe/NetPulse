🌐 NetPulse — Утилита для управления DNS и проверки доступности сайтов
Зачем нужна ⚙️
NetPulse меняет DNS-серверы в Windows на быстрые публичные (Google, Cloudflare, Yandex) и проверяет, какие сайты доступны, а какие заблокированы.
Что делает ✅

Ускоряет интернет — заменяет DNS провайдера на более быстрые, страницы начинают открываться быстрее.
Обходит простые блокировки — если провайдер подменяет адреса сайтов, NetPulse возвращает настоящие IP-адреса.
Диагностирует проблемы — проверяет доступность сайтов через TCP (порт 443), это ближе к реальному HTTPS-соединению, чем обычный пинг, и обходит блокировку ICMP брандмауэрами.

Как понять тип блокировки ⚠️

 DNS-блокировка — сайт не открывается, но после смены DNS заработал. Проблема была в DNS-сервере провайдера.
DPI (глубокая проверка пакетов) — соединение есть, но контент не грузится. Провайдер анализирует содержимое TLS-пакетов и обрывает соединение, смена DNS не поможет.
Блокировка по IP/порту — полный отказ соединения при корректном IP-адресе. Провайдер блокирует на уровне маршрутизатора.

Как работает 🛠️
Написана на Go. Находит активный сетевой адаптер (игнорируя VPN и виртуальные), меняет DNS через netsh и параллельно проверяет список сайтов.
Полезные команды для проверки 💻
plain

ipconfig /all        — посмотреть текущие DNS-серверы
nslookup <сайт>      — проверить, какой IP выдаёт DNS
ipconfig /flushdns   — сбросить кэш DNS
tracert <сайт>       — увидеть, на каком узле обрывается связь

Итог: NetPulse помогает ускорить интернет, обойти простые блокировки и понять, как именно провайдер фильтрует трафик. 🚀
