🌐 NetPulse — Инструкция по использованию

NetPulse — это инструмент для управления DNS-серверами Windows и диагностики доступности сети. Он позволяет быстро переключаться между разными DNS-провайдерами и проверять, какие ресурсы доступны, а какие блокируются.
🚀 Зачем использовать NetPulse? (Полезность)

    Ускорение интернета: DNS провайдера часто медленный. Переключившись на Cloudflare или Google, вы уменьшаете задержку при первом открытии сайтов.
    Обход простых блокировок: Помогает открыть сайты, которые провайдер блокирует простым «подменом» IP-адреса (DNS-спуфинг).
    Поиск лучшего сервера: С помощью встроенного теста можно найти, какой DNS-сервер в вашем городе работает быстрее всего.
    Диагностика сети: Вы можете точно понять, почему сайт не открывается: из-за того, что DNS не может найти адрес, или из-за того, что провайдер блокирует сам трафик.

⚠️ Что ТОЧНО не заработает через DNS?

Важно понимать: DNS — это просто «адресная книга». Смена DNS не поможет, если провайдер использует DPI (Deep Packet Inspection).

    YouTube (замедление/блокировка): Не починится сменой DNS, так как режется сам видео-поток.
    Жесткие блокировки (например, Telegram): Если IP-адреса сервиса внесены в «черный список» провайдера, смена DNS не поможет — пакеты просто не пройдут.
    Региональные ограничения: Если сайт доступен только в США, DNS не сделает вас «американцем» (для этого нужен VPN/Proxy).

🛠 Команды и управление
1. Запуск программы

Программу ОБЯЗАТЕЛЬНО запускать от имени Администратора, так как изменение сетевых настроек Windows требует повышенных прав.

Запуск основного меню (TUI):

go run main.go

Запуск теста доступности ресурсов (Пинг):

netpulse ping

2. Полезные системные команды (вне программы)

Если вы хотите проверить сеть вручную через стандартную консоль Windows (cmd), используйте эти команды:
Проверка текущих DNS и адаптера:

Показывает все настройки сети, включая текущие DNS-серверы.

ipconfig /all

Проверка, какой IP выдает DNS:

Позволяет узнать, какой IP-адрес «видит» система для конкретного сайта.

nslookup google.com

(Если команда выдает странный IP или ошибку — значит, DNS работает плохо).
Обычный ICMP-пинг (проверка связи с сервером):

Проверяет, отвечает ли сервер в принципе (не все серверы разрешают такой пинг).

ping 8.8.8.8

Очистка кэша DNS:

После смены DNS в программе рекомендуется очистить кэш Windows, чтобы изменения вступили в силу мгновенно.

ipconfig /flushdns

Трассировка маршрута:

Показывает, через какие узлы (города/провайдеров) проходит ваш запрос до сайта. Помогает понять, на каком этапе обрывается связь.

tracert youtube.com

Краткий итог:

    DNS-чейнджер → для скорости, стабильности и обхода простых блокировок.
    VPN/Прокси → для обхода DPI и региональных запретов (YouTube, Telegram).
