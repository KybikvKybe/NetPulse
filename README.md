🌐 NetPulse — Системная утилита для управления DNS-конфигурацией и анализа доступности сетевых ресурсов

Техническое назначение ⚙️
NetPulse представляет собой инструмент для автоматизации управления параметрами IPv4-интерфейсов в ОС Windows. Основной функционал сосредоточен на модификации настроек DNS-резолверов через системный интерфейс netsh и проведении комплексного анализа доступности удаленных хостов с использованием TCP-проверки.

Функциональные возможности и профит ✅

    Оптимизация DNS-резолвинга: Замена стандартных DNS-серверов провайдера на публичные высокопроизводительные рекурсоры (Google, Cloudflare, Yandex). Это позволяет снизить задержку (latency) при этапе DNS-запроса (DNS Lookup) и избежать проблем с некорректными или устаревшими записями в кэше провайдера.
    Обход DNS-спуфинга: Инструмент позволяет обходить примитивные методы цензуры, основанные на подмене IP-адресов (DNS Poisoning), обеспечивая получение истинных A-записей доменов.
    Сетевой аудит (Diagnostic Mode): Модуль ping реализует проверку доступности ресурсов через TCP-соединение (порт 443). В отличие от классического ICMP-пинга, который часто блокируется брандмауэрами, TCP-проверка имитирует реальный HTTPS-запрос, что дает более точную картину доступности сервиса.

Ограничения и анализ блокировок ⚠️
Программа позволяет четко дифференцировать тип блокировки ресурса:

    DNS-блокировка: Если ресурс не резолвится (Resolve Failed), но после смены DNS начинает пинговаться → проблема была в DNS-сервере.
    L7-блокировка (DPI): Если TCP-соединение устанавливается (статус ✅), но данные не передаются (например, видео на YouTube не грузится), это свидетельствует о работе систем глубокого анализа пакетов (Deep Packet Inspection). DPI анализирует содержимое TLS-пакетов и обрывает соединение на уровне протокола, что делает смену DNS бесполезной.
    L3/L4-блокировка: Статус UNREACHABLE при корректном IP-адресе указывает на блокировку по IP-адресу или порту на уровне маршрутизатора провайдера.

Технический стек и реализация 🛠️

    Язык: Go (Golang).
    Метод управления: Взаимодействие с Windows API через вызовы командной строки netsh interface ip.
    Поиск адаптера: Динамическое определение активного физического сетевого интерфейса с фильтрацией виртуальных адаптеров (VPN, VMware, VirtualBox) по маскам имен и флагам net.FlagUp.
    Диагностика: Параллельный опрос списка хостов с использованием net.DialTimeout для минимизации времени ожидания ответа.

Системные команды для верификации 💻
Для дополнительного анализа сетевого стека рекомендуется использовать:
👉 ipconfig /all — анализ текущей конфигурации TCP/IP и активных DNS-серверов.
👉 nslookup <domain> — проверка работы конкретного резолвера.
👉 ipconfig /flushdns — очистка системного кэша DNS для исключения влияния старых записей.
👉 tracert <domain> — анализ маршрута прохождения пакетов и определение узла, на котором происходит сброс соединения.

Резюме: NetPulse является инструментом низкоуровневого управления сетевыми настройками, позволяющим оптимизировать скорость доступа к сети и проводить базовый анализ методов фильтрации трафика провайдером. 🚀🌐
